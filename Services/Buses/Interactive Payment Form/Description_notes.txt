SLIDE 6: VALIDATION SYSTEM - WHY IT MATTERS
================================================================================
WHAT IS VALIDATION?
Validation means checking if user input is correct before processing.

WHY IS IT IMPORTANT?
✓ Prevents errors in payment processing
✓ Protects against invalid data
✓ Improves security
✓ Saves time for users and businesses
✓ Reduces failed transactions
✓ Better user experience

OUR VALIDATION CHECKS:
1. Card Number: Must be exactly 16 digits
2. Cardholder Name: Cannot be empty
3. Expiry Month: Must be selected
4. Expiry Year: Must be selected
5. CVV: Must be exactly 3 digits

================================================================================
SLIDE 7: HOW VALIDATION WORKS IN OUR CODE
================================================================================
VALIDATION PROCESS:

STEP 1: User fills the form
STEP 2: User clicks "Submit"
STEP 3: JavaScript checks each field:

CODE EXAMPLE:
--------------
if (cardNum.length < 16) {
    alert('Please enter valid card number');
    return; // Stop here, don't continue
}

if (!holder) {
    alert('Please enter cardholder name');
    return;
}

if (cvv.length !== 3) {
    alert('CVV must be 3 digits');
    return;
}

STEP 4: If all checks pass → Success message
STEP 5: If any check fails → Error message and focus on that field

7. UNIONPAY
   - Starts with: 62

SLIDE 11: CODE STRUCTURE - JAVASCRIPT
================================================================================
JAVASCRIPT FILE (my.js):
Purpose: Makes the form interactive

KEY FUNCTIONS:

1. CARD TYPE DETECTION:
   function detectCardType(number) {
       if (/^4/.test(number)) return 'visa';
       if (/^5[1-5]/.test(number)) return 'mastercard';
       // ... more checks
   }
   
   → Checks first digits of card number
   → Returns card type name
   → Used to show correct logo

2. NUMBER FORMATTING:
   let formatted = value.match(/.{1,4}/g)?.join(' ');
   
   → Splits number into groups of 4
   → Adds spaces between groups
   → Example: "1234567890123456" → "1234 5678 9012 3456"

3. REAL-TIME UPDATE:
   cardNumberInput.addEventListener('input', function(e) {
       cardNumberDisplay.textContent = formattedValue;
   });
   
   → Listens for user typing
   → Updates card display instantly
   → Creates live preview effect

4. CARD FLIP:
   cvvInput.addEventListener('focus', function() {
       card.classList.add('flipped');
   });
   
   → Triggers when CVV field is clicked
   → Adds 'flipped' class to card
   → CSS rotates card to show back

5. VALIDATION:
   if (cardNum.length < 16) {
       alert('Invalid card number');
       return;
   }
   
   → Checks data before submission
   → Shows error if invalid
   → Prevents form submission

SLIDE 12: MOST IMPORTANT CODE SECTIONS
================================================================================

SECTION 1: AUTO-FORMATTING CARD NUMBER
--------------------------------------
cardNumberInput.addEventListener('input', function(e) {
    let value = e.target.value.replace(/\s/g, '');
    let formatted = value.match(/.{1,4}/g)?.join(' ') || value;
    e.target.value = formatted;
    cardNumberDisplay.textContent = formatted || '#### #### #### ####';
});

WHAT IT DOES:
- Removes all spaces from input
- Groups digits in sets of 4
- Joins groups with spaces
- Updates both input field and card display

--------------------------------------

SECTION 2: CARD TYPE DETECTION & LOGO DISPLAY
--------------------------------------
const cardType = detectCardType(value);
if (cardType && cardLogos[cardType]) {
    cardLogo.innerHTML = cardLogos[cardType];
}

WHAT IT DOES:
- Detects card brand from number
- Finds matching logo in cardLogos object
- Displays SVG logo on card

--------------------------------------

SECTION 3: 3D CARD FLIP ANIMATION
--------------------------------------
.card.flipped {
    transform: rotateY(180deg);
}

cvvInput.addEventListener('focus', function() {
    card.classList.add('flipped');
});

WHAT IT DOES:
- CSS defines flip animation
- JavaScript triggers flip when CVV is focused
- Card rotates to show back side

--------------------------------------

SECTION 4: COMPLETE FORM VALIDATION
--------------------------------------
form.addEventListener('submit', function(e) {
    e.preventDefault();
    
    if (cardNum.length < 16) {
        alert('Invalid card number');
        cardNumberInput.focus();
        return;
    }
    
    if (!holder) {
        alert('Enter cardholder name');
        return;
    }
    
    // More validation checks...
    
    alert('Payment submitted successfully!');
});

WHAT IT DOES:
- Prevents default form submission
- Checks each field one by one
- Shows specific error message
- Focuses on problem field
- Only continues if all valid
